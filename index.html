If a free Pollinations request is busy, AdForge automatically falls back to Gemini for the text steps (a separate free tier). Images are independent, always free via Pollinations.

Pollinations is keyless, so no proxy is required. To offer the Gemini fallback to public visitors without their own key, deploy to Vercel, set GEMINI_API_KEY, and set this to /api (the api/gemini.js proxy holds the key server-side).

Leave blank to use the in-browser recorder. Paste your deployed adforge-render URL to get frame-accurate GSAP video via HyperFrames (see that repo's README to deploy in a few clicks).

How AdForge works

AdForge runs a team of five AI agents in a pipeline, then loops a critic over the results:

  1. Strategist: turns your idea into a creative brief (angle, audience, message, mood, palette).
  2. Prompt Engineer: expands the brief into a strong master image prompt. Also powers Improve prompt.
  3. Art Director: splits the campaign into distinct scenes (hero, lifestyle, detail, CTA…).
  4. Copywriter: writes headline, subhead, body, CTA and a social caption per scene.
  5. Critic: scores each scene against the brief and proposes one concrete fix you can apply in a click.

You can edit the master prompt (regenerates every scene) or any single scene's prompt (regenerates just that image), and rewrite copy or apply the critic's fix per scene.

Stack: vanilla JS · free & keyless. The writing agents run on Pollinations (with an optional Gemini fallback) and images render via Pollinations (FLUX, with a fast turbo fallback). No backend required (an optional Vercel proxy can hold a Gemini key for public visitors).
