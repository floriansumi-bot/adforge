Settings

Images are free, no key needed — they render via Pollinations (FLUX). The free Z.ai key below powers the GLM writing agents. Your key is stored only in this browser and is sent only to Z.ai.

Get a free key → z.ai (GLM-4.7-Flash is free) Get a free key → Google AI Studio

When Z.ai's free GLM is overloaded, AdForge falls back to Gemini for the agent text steps (images are independent — free via Pollinations). A separate free tier, so it usually isn't busy at the same time.

For a public demo where visitors need no key, deploy to Vercel, set the ZAI_API_KEY env var, and set this to /api. The proxy (api/glm.js, api/image.js) then holds the key server-side. On GitHub Pages, leave this blank and each visitor adds their own free key.

Leave blank to use the in-browser recorder. Paste your deployed adforge-render URL to unlock frame-accurate GSAP video via HyperFrames (see that repo's README to deploy in a few clicks).

How AdForge works

AdForge runs a team of five AI agents in a pipeline, then loops a critic over the results:

  1. Strategist — turns your idea into a creative brief (angle, audience, message, mood, palette).
  2. Prompt Engineer — expands the brief into a strong master image prompt. Also powers Improve prompt.
  3. Art Director — splits the campaign into distinct scenes (hero, lifestyle, detail, CTA…).
  4. Copywriter — writes headline, subhead, body, CTA and a social caption per scene.
  5. Critic — scores each scene against the brief and proposes one concrete fix you can apply in a click.

You can edit the master prompt (regenerates every scene) or any single scene's prompt (regenerates just that image), and rewrite copy or apply the critic's fix per scene.

Stack: vanilla JS · one free Z.ai key powers everything — GLM-4.7-Flash / GLM-5.1 for the agents and CogView-3-Flash for the images. No backend required (an optional Vercel proxy can hold the key for public visitors).
